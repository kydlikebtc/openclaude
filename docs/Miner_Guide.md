## 一、什么是 OpenClade Miner？

OpenClade 是基于 **Bittensor TAO 子网**的分布式 Claude API 服务平台。用户支付极低价格（官方 25-35%）使用 Claude，而 **Miner（矿工）** 通过提供 Claude API Key 来处理用户请求，获得 **TAO 子网 Emission（挖矿奖励）**。

### 工作原理

```
用户 ──请求──→ OpenClade 路由层 ──→ Miner (你的 API Key) ──→ Claude API
                                                ↓
                                    TAO Emission 奖励 (每日结算)
```

作为 Miner，你：
- **提供**：Claude API Key（来自你的 Anthropic 账号）
- **运行**：在线服务节点，处理用户的 Claude API 请求
- **获得**：根据服务质量分配的 TAO 子网 Emission

**核心机制：** TAO Emission 奖励 > API 成本，Miner 纯利润空间极大。

## 二、为什么要成为 Miner？ROI 分析

### 2.1 盈利模型

Miner 的经济逻辑非常简单：

```
TAO Emission 收入 - Claude API 成本 = 净利润
```

在正常情况下，TAO 挖矿收入远高于 API 调用成本。

### 2.2 收益测算（基准场景）

**假设条件：**
- 子网日 Emission: 50 TAO
- TAO 市场价格: $300
- Miner 总数: 50 个

| 场景 | 日处理 Token | 在线率 | 探测成功率 | 推荐加成 | 日 TAO 收入 | API 成本 | **日净利润** |
|------|-------------|--------|------------|---------|------------|---------|-------------|
| 新手 Miner | 2M tokens | 90% | 90% | 0% | 0.41 TAO | $12 | **$111** |
| 成熟 Miner | 5M tokens | 99% | 99% | 12% | 1.38 TAO | $30 | **$384** |
| 顶级 Miner | 10M tokens | 99% | 99% | 30% | 2.9 TAO | $60 | **$810** |

**月利润预估（按日净利润 × 30）：**
- 新手：~$3,300/月
- 成熟：~$11,520/月
- 顶级：~$24,300/月

### 2.3 风险缓冲

**TAO 大幅下跌时仍可盈利：**

| TAO 价格 | 新手日 TAO 收入 | API 成本 | 状态 |
|----------|----------------|---------|------|
| $300 | $123 | $12 | ✅ 净利 $111 |
| $150 | $61.5 | $12 | ✅ 净利 $49.5 |
| $50 | $20.5 | $12 | ✅ 净利 $8.5 |
| $29 | $11.9 | $12 | ⚠️ 盈亏平衡 |

> **结论：** 即使 TAO 价格从 $300 跌至 $30（跌幅 90%），Miner 仍接近盈亏平衡，远比许多传统挖矿稳健。

## 三、前置条件

在开始之前，请确认你具备以下条件：

### 必须具备

| 条件 | 说明 |
|------|------|
| ✅ **Claude API Key** | Anthropic 官方账号的 API Key（`sk-ant-api03-...`） |
| ✅ **TAO 钱包** | Bittensor 兼容钱包（如 Polkadot.js 扩展、Bittensor CLI） |
| ✅ **5 TAO 质押** | 最低质押要求，约 $1,500（@$300/TAO） |
| ✅ **稳定网络** | 服务器或稳定网络，保证在线率 ≥ 80% |

### 推荐配置

| 配置项 | 推荐 | 最低 |
|--------|------|------|
| API Key 余额 | $50+（避免频繁充值） | $10 |
| TAO 质押量 | 10-20 TAO（更高得分） | 5 TAO |
| 在线率目标 | ≥ 95%（最大化收益） | 80% |
| 探测响应时间 | ≤ 1,500ms | ≤ 3,000ms |

### 获取 Claude API Key

1. 访问 [console.anthropic.com](https://console.anthropic.com)
2. 注册/登录 Anthropic 账号
3. 进入 **API Keys** 页面创建新 Key
4. 建议充值 $50-100 作为初始余额

> ⚠️ **注意：** 提交到 OpenClade 的 API Key 需要有足够的账单余额，否则探测请求会失败，导致得分为 0。

## 四、注册与上手流程

### 步骤一：注册 OpenClade 账号

1. 访问 **[openclaude.io](https://openclaude.io)**
2. 点击 **"成为 Miner"** 或 **"Miner 注册"**
3. 使用邮箱完成注册

### 步骤二：连接 TAO 钱包

1. 登录后，进入 **Miner 控制台**
2. 点击 **"连接钱包"**
3. 系统会展示一段签名消息（Challenge），格式如下：
   ```
   Sign in to OpenClade: abc123def456
   ```
4. 使用你的 TAO hotkey 对该消息进行 SR25519 签名
5. 提交签名完成钱包验证

**使用 Bittensor CLI 签名：**
```bash
# 查看你的 hotkey 地址
btcli wallet overview

# 生成签名（根据 UI 中的 challenge 替换下面的消息）
btcli wallet sign --message "Sign in to OpenClade: abc123def456" --wallet.name default
```

### 步骤三：质押 TAO

1. 在 Miner 控制台找到 **"质押管理"**
2. 查看你的钱包地址（hotkey）
3. 通过 Bittensor CLI 或 Polkadot.js 转入至少 **5 TAO**：
   ```bash
   btcli stake add --hotkey <your-hotkey> --amount 5
   ```
4. 等待链上确认（约 12 秒一个区块）
5. 刷新控制台确认质押状态显示 `✅ 已质押: X.X TAO`

### 步骤四：提交 Claude API Key

1. 进入 Miner 控制台 → **"API Keys 管理"**
2. 点击 **"添加 API Key"**
3. 粘贴你的 Claude API Key（格式：`sk-ant-api03-...`）
4. 系统自动验证 Key 可用性（约 5-10 秒）
5. 验证成功后，Key 状态变为 `✅ 已激活`

> ⚠️ **安全提示：** 你的 Key 在系统中会被加密存储，界面只展示前后几位（如 `sk-ant-...abcd`）。OpenClade 团队无法查看完整 Key 内容。

### 步骤五：确认节点上线

1. 返回 Miner 控制台首页
2. 检查 **"服务质量"** 面板，确认：
   - 在线率 ≥ 80% ✅
   - 探测成功率 ≥ 90% ✅
   - 平均延迟 ≤ 3,000ms ✅
   - 质押状态 ≥ 5 TAO ✅
3. 出现 **"准入资格：已获得"** 提示后，你的节点开始参与评分
4. 首次评分结算在下一个 Epoch 开始（通常每小时）

## 五、收益机制详解

### 5.1 TAO Emission 分配结构

每日子网产生的 TAO Emission 按以下比例分配：

```
子网日 Emission
├── 18% → OpenClade 运营方（平台）
├── 41% → 全体 Miners（按评分比例分配）
└── 41% → Validators（验证节点）
```

### 5.2 准入门槛

**以下四项条件必须全部满足**，否则本 Epoch 得分 = 0（不参与 Emission 分配）：

| 指标 | 最低要求 | 说明 |
|------|----------|------|
| 探测成功率 | ≥ 90% | Validator 发送测试请求，你的 Key 必须正确响应 |
| 在线率 | ≥ 80% | 过去 24 小时内的在线时长占比 |
| 平均延迟 | ≤ 3,000ms | 端到端响应时间（含 Claude API 调用） |
| 最低质押 | ≥ 5 TAO | 防止垃圾节点的经济门槛 |

### 5.3 评分公式

```
最终得分 = 服务分 × (1 + 推荐加成)
```

**服务分 = 贡献分 × W1 + 待命分 × W2**

| 分项 | 计算方式 | 说明 |
|------|---------|------|
| **贡献分** | (你处理的 Token / 全网 Token) × 100 | 实际为网络贡献了多少 |
| **待命分** | 在线率×50 + 探测成功率×50 | 即使流量少也能赚基础收益 |

**动态权重（W1/W2）：**

| 网络日 Token | W1（贡献权重） | W2（待命权重） |
|-------------|--------------|--------------|
| < 10M | 30% | 70% |
| 10M - 500M | 线性插值 | 线性插值 |
| > 500M | 80% | 20% |

> **冷启动阶段（早期流量少）：** 待命分权重高达 70%，意味着只要保持高在线率，即使没有流量也能获得稳定收益。

### 5.4 推荐加成

通过你的**专属推荐码**引入用户，可在服务分基础上获得额外加成：

```
推荐加成 = min(你的推荐用户消费占全网 × 2 × 30%, 30%)
```

**示例：**

| 你的推荐用户本期消费 | 全网消费 | 占比 | 推荐加成 |
|---------------------|---------|------|---------|
| $0 | $10,000 | 0% | 0% |
| $500 | $10,000 | 5% | 3% |
| $2,500 | $10,000 | 25% | 15% |
| $6,000 | $10,000 | 60% | **30%（上限）** |

**规则：**
- 推荐用户**本期消费 ≥ $5** 才计入（防刷量）
- 只统计**当期**消费，不积累历史
- 推荐加成最高 **30%**（服务分倍增器，锦上添花）
- 推荐关系**永久绑定**，不可更改

### 5.5 推荐码使用

每个 Miner 自动获得唯一推荐码（格式：`OC-XXXXXXXX`），以及专属推荐链接：
```
https://openclaude.io/r/OC-A1B2C3D4
```

## 七、常见问题 FAQ

**Q1: 我需要运行任何服务器吗？**

A: 不需要。OpenClade 是一个平台，你只需要在 Web 控制台提交你的 API Key，平台自动路由用户请求到你的 Key。你不需要部署任何代码或服务器（未来可能有可选的自托管模式）。

---

**Q2: 我的 API Key 安全吗？**

A: 系统中 API Key 以加密形式存储，界面只展示首尾几位（如 `sk-ant-...abcd`）。但请注意：
- Key 提交后会被平台用于处理用户请求
- 这意味着 Key 的调用量和使用模式可能触发 Anthropic 的风控
- 建议使用专门的账号和 Key 用于 Mining，与个人主要账号分开

---

**Q3: 如果我的 API Key 被 Anthropic 封禁怎么办？**

A:
1. 登录 Miner 控制台，将该 Key 状态设为 "禁用"
2. 申请新的 Anthropic 账号和 Key
3. 在控制台添加新 Key
4. 封禁期间你的 Miner 节点可能因探测失败率上升而暂时失去资格

---

**Q4: 如何最大化我的收益？**

A: 按优先级：
1. **维持高在线率（≥ 95%）**：早期流量少，待命分权重高，在线率是核心
2. **API Key 余额充足**：Key 余额不足会导致探测失败，直接影响准入资格
3. **积极推广推荐码**：推荐加成最高可使最终得分提升 30%
4. **增加 TAO 质押（超过 5 TAO）**：降低质押不足的风险

---

**Q5: 评分多久结算一次？**

A: Epoch 周期为每小时，但具体结算周期取决于 Bittensor 网络参数。TAO Emission 每日汇总分配。Miner 控制台会实时显示当前预估收益。

---

**Q6: 我的推荐码在哪里？**

A: 登录 Miner 控制台 → **"推荐系统"** 页面，你会看到：
- 你的专属推荐码（`OC-XXXXXXXX` 格式）
- 推荐链接（`https://openclaude.io/r/OC-XXXXXXXX`）
- 推荐统计（用户数、当期消费、推荐加成比例）

---

**Q7: 推荐码给用户有什么好处？**

A: 使用推荐码注册的用户可享受：
- **首充额外赠送 10%**（充 $100 得 $110）
- **前 30 天额外 95 折优惠**

这对用户很有吸引力，能显著提升你的推广转化率。

---

**Q8: 多个 Miner 能否使用同一个 API Key？**

A: 不建议。每个 API Key 都有 Anthropic 端的速率限制，多个节点共用一个 Key 可能导致请求失败率上升，影响探测成功率。建议每个 Miner 账号使用独立的 API Key。

---

**Q9: 何时是加入的最佳时机？**

A: **现在！** 原因：
- **冷启动加成**：早期（前 3 个月，Miner 数量 < 20 个）参与的 Miner 有额外 30-50% 的加成
- **早鸟优势**：Miner 数量越少，每个 Miner 的 Emission 份额越大
- **推荐先机**：早期加入的 Miner 更容易建立大的推荐网络

---

**Q10: 如何查看我的实时排名？**

A: Miner 控制台的 **"收益总览"** 页面会显示：
- 当日得分
- 当日处理 Token 量
- 预估 TAO 收益
- **你在全网 Miner 中的排名**（如 "第 5 / 23"）
