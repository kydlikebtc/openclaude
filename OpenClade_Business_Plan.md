## 二、商业故事

### 2.1 问题：AI API 成本之痛

#### Claude API 官方定价 (以 Sonnet 为例)

| 类型 | 价格 |
|------|------|
| Input | $3 / 百万 tokens |
| Output | $15 / 百万 tokens |

#### 典型用户月成本

| 用户类型 | 月成本 |
|----------|--------|
| 个人开发者 | $50-200/月 |
| 小型团队 | $500-2,000/月 |
| 中型企业 | $5,000-20,000/月 |

#### 核心痛点

- 高成本限制了 AI 应用的普及
- 个人开发者和初创公司难以承受
- 教育和研究领域使用受限
- 很多创新想法因成本而夭折

### 2.2 解决方案：TAO 子网驱动的成本革命

#### 核心创新

利用 TAO 子网的挖矿激励机制，补贴 API 使用成本

#### 模式对比

**传统模式：**
```
用户 ──$100──→ 服务商 ──$100──→ Claude API
```

**OpenClade 模式：**
```
用户 ──$25──→ 平台 ──路由──→ Miner ──$100──→ Claude API
                ↑                  ↑
                │                  │
             回购代币         TAO Emission
                │                  │
                └────── TAO 网络 ──┘
```

**关键点：** Miner 的成本由 TAO 挖矿奖励覆盖，用户只需支付极低价格

#### 用户获益

- 官方价格 $100 → OpenClade 价格 $25-35
- 节省 65-75%
- 完全兼容官方 API 格式

### 2.3 为什么是现在？

#### 1. TAO 生态成熟
- Bittensor 子网机制稳定运行
- TAO 代币有稳定的价值基础
- 子网经济模型被验证可行

#### 2. AI API 需求爆发
- ChatGPT/Claude 引爆市场
- 开发者和企业大量采用 AI
- 成本敏感度持续上升

#### 3. 竞争空白
- 目前没有成熟的低价 Claude API 服务
- TAO 子网在 AI API 领域应用较少
- 先发优势窗口存在

## 五、产品机制设计

### 5.1 Miner 评分机制

#### 核心公式

```
最终得分 = 服务分 × (1 + 推荐加成)

前提: 必须通过准入门槛
```

#### 准入门槛

必须全部满足，否则该 Epoch 得分 = 0：

| 指标 | 要求 |
|------|------|
| 探测成功率 | ≥ 90% |
| 在线率 | ≥ 80% |
| 平均延迟 | ≤ 3 秒 |
| 质押 | ≥ 5 TAO |

#### 服务分 (核心，占主导)

```
服务分 = 贡献分 × W1 + 待命分 × W2
```

**贡献分：**
- 实际处理 Token 量占全网比例 × 100

**待命分：**
- 在线率 × 50 + 探测成功率 × 50

**动态权重 (根据网络使用量)：**

| 网络使用量 | W1 (贡献分) | W2 (待命分) |
|------------|-------------|-------------|
| 低 (<10M/天) | 30% | 70% |
| 高 (>500M/天) | 80% | 20% |

#### 推荐加成 (添头，非主导)

```
推荐加成 = min(当期推荐消费占比 × 2 × 30%, 30%)
```

**关键规则：**
- 只看当期消费，非累计（避免分母膨胀）
- 只看消费金额，不看用户数（激励拉活跃用户）
- 用户消费 ≥ $5 才计入（防刷量）
- 上限 30%（防止喧宾夺主）
- 推荐关系永久绑定（防恶意竞争）

**设计原理：**
- 服务分 = 0 → 最终得分 = 0（无论推荐多少）
- 推荐是乘数加成，锦上添花而非雪中送炭
- 激励 Miner 主动拉用户，但不能靠拉人替代服务能力

### 5.2 推荐机制详解

#### 机制目标

把 Miner 变成分销渠道，用 TAO emission 激励拉客

**传统模式：**
```
平台花钱推广 → 获取用户
```

**OpenClade 模式：**
```
Miner 为了更多 TAO → 主动拉用户 → 平台白嫖分销网络
```

#### 推荐码机制

每个 Miner 自动获得唯一推荐码：`OC-XXXXXXXX`

**用户注册时看到：**

```
🎉 使用推荐码注册，享受专属福利！

推荐码: [OC-A1B2C3D4] (选填)

福利:
├── ✅ 首次充值额外赠送 10%
└── ✅ 前 30 天额外 95 折优惠
```

#### 防刷量机制

| 措施 | 说明 |
|------|------|
| 消费门槛 | 用户消费 ≥ $5 才计入推荐 |
| 真实调用成本 | 刷量需要真实 API 调用，Miner 自己付成本 |
| 推荐关系绑定 | 永久绑定，不可更改 |
| 作弊惩罚 | 检测到刷量 → 罚没质押 |

#### 内卷飞轮效应

```
Miner 为了更多 emission → 主动推广
          │
          ├── 社交媒体推广 (Twitter/Discord)
          ├── 技术社区分享 (GitHub/论坛)
          └── KOL 合作 (YouTube 等)
                   │
                   ▼
用户爆发增长 → Token 消耗↑ → 价值证明↑ → Emission↑
          │
          └── Miner 收益↑ → 更多 Miner 加入内卷拉客
```

### 5.3 Validator 机制

#### 核心职责

**1. 实时探测 (Probing)**
- 每分钟探测所有 Miner
- 发送随机测试请求
- 记录在线状态、延迟、成功率

**2. 响应验证 (Verification)**
- 验证响应是否真实来自 Claude
- 检测伪造响应
- 检测模型降级（声称 Opus 实际用 Haiku）

**3. 评分提交 (Scoring)**
- 汇总各 Miner 的质量数据
- 计算评分
- 提交到链上

#### 探测机制设计

```javascript
// 随机数学问题（防缓存，易验证）
prompt: "Calculate 847 × 293. Only output the number."

// 验证:
// - 响应是否正确 (248171)
// - 响应时间
// - 响应格式是否符合 Claude API 规范
```

| 在线率 | 99% |
| 探测成功率 | 99% |
| 推荐用户消费 | 占全网 20% |
| 服务分占比 | ~6% |
| 推荐加成 | 12% |
| 最终占比 | 6% × 1.12 = 6.72% |
| 日 TAO | 20.5 × 6.72% = 1.38 TAO |
| 日收入 | $414 |
| API 成本 | 5M × $6/M = $30 |
| **日净利润** | **$384** |
| **月净利润** | **$11,520** |

#### Miner 盈亏平衡分析

只要 TAO 奖励价值 > API 成本，Miner 就盈利

以普通 Miner 为例:
- TAO 奖励: 0.41 TAO = $123
- API 成本: $12
- **利润率: 925%**

**TAO 价格跌到多少 Miner 才亏损？**
- 盈亏平衡: 0.41 TAO × X = $12
- X = $29.3/TAO
- **即 TAO 跌 90%，Miner 仍不亏损**

## 八、风险分析与应对

### 8.1 风险矩阵

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| Anthropic 封禁 API Key | 高 | 高 | Key 来源分散，快速替换机制，多 Miner 冗余 |
| TAO 价格暴跌 | 中 | 高 | 50% 收入转稳定币储备，保持 6 个月 runway |
| 用户增长不及预期 | 中 | 中 | 超低价策略，Miner 推荐机制，社区运营 |
| 竞争对手出现 | 中 | 中 | 先发优势，网络效应，持续迭代 |
| 法律/合规问题 | 中 | 高 | 法律架构设计，管辖区选择，合规咨询 |
| 技术故障/攻击 | 低 | 中 | 多重冗余，安全审计，监控告警 |

### 8.2 核心风险详细分析

#### 风险 1: API Key 封禁

**问题：**
- Anthropic 检测到异常使用模式
- 批量封禁 Key
- 服务中断

**应对：**
- 多 Miner 设计: 单个 Key 封禁影响有限
- Key 来源分散: 不同账号、不同地区
- 使用模式正常化: 避免明显的异常特征
- 快速替换机制: 检测到封禁立即切换
- 用户教育: 合理使用，避免滥用

#### 风险 2: TAO 价格波动

**问题：**
- Owner 收入完全以 TAO 计价
- Miner 成本以 USD 计价
- 价格下跌可能导致生态崩溃

**应对：**
- 收入管理: 50% TAO 立即转稳定币
- 储备金: 保持 6 个月运营 runway
- 成本控制: 最小化固定成本
- 多元收入: 用户付费作为补充

#### 风险 3: 合规风险

**问题：**
- API 转售可能违反 ToS
- 法律责任不明确
- 监管不确定性

**应对：**
- 法律架构: 选择友好管辖区
- 去中心化: 平台不直接持有 Key
- 用户协议: 明确责任边界
- 合规咨询: 定期法律评估

### B. 核心公式汇总

#### Miner 评分

```
最终得分 = 服务分 × (1 + 推荐加成)

服务分 = 贡献分 × W1 + 待命分 × W2
- 贡献分 = 实际处理 Token 占比 × 100
- 待命分 = 在线率 × 50 + 探测成功率 × 50
- 动态权重: 低使用量 W1=30%/W2=70%, 高使用量 W1=80%/W2=20%

推荐加成 = min(推荐消费占比 × 2 × 30%, 30%)
```

#### 收益计算

```
Owner 日收入 = 日 Emission × 18% × TAO 价格

Miner 日收入 = 日 Emission × 41% × (Miner 得分/全网总分) × TAO 价格
Miner 日成本 = 处理 Token 量 × Claude 官方价格
Miner 日利润 = 日收入 - 日成本
```

#### 平台经济

```
日收入 = 日 Token 消耗 × 平台定价
月运营成本 = 固定成本 + 日 Token × 30 × 可变成本率
月盈余 = 月收入 - 月运营成本
回购金额 = 月收入 - 月运营成本 (盈余全部回购)
```
